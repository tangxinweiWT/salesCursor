# 实战篇
1、描述需求
描述——具体清晰
说明——项目背景和目标
指出——技术栈和框架版本
提供——具体功能描述和预期结果
说明——性能要求和限制条件

2、步骤引导
将开发任务分为小步骤，逐步让cursor完成。
基础框架 → 具体功能 → 优化完善

3、上下文
代码片段，错误信息，尝试过的方法，约束条件

4、迭代优化


5、有效反馈
确认是否满足需求，提供实际运行结果

>> 模式
ask，agent，manual
ask 询问 （ai 回答）(在构建项目时候可以先使用ask进行项目构建让ai给出方案以便修改，之后使用agent进行项目构建 )
agent 代理 （由 ai 代理进行开发）
manual 精细（manual 之后对你选中的代码进行修改）

>> notepad 功能
右键资源管理器，选择允许创建notepad，可以创建你的笔记（可以记录些你的想法，或者你想要的规定）
在询问chat 时可以@ notepad引用你的笔记作为上下文，引导ai

让ai读懂文档的方式
1、@引用
2、file
3、doc

doc
把文档添加到cursor中，让他先熟悉，它可以做的更好
逻辑为cursor会把文档向量化，记得去刷新文档，不然文档更新cursor是不知道的
